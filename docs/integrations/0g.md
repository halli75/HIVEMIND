# 0G Integration

## Targets

- 0G Compute for active inference.
- 0G Storage for agent state, memory, and run logs.
- 0G Chain for the ERC-721-compatible iNFT mint on Galileo testnet.

## Current Status

- Contract network config points at 0G Galileo testnet by default through `ZERO_G_RPC_URL`.
- Live 0G Compute is wired behind `ZeroGComputeInferenceProvider` and the hybrid top-N inference path.
- `contracts/scripts/mint-inft.ts` fetches the current winner, encrypts the strategy with AES-256-GCM, uploads to 0G Storage, then mints `HivemindINFT` on Galileo.
- `/mint` delegates to that script, returns structured storage errors before minting, and updates `/state.proof.inft` after a successful mint.
- `HivemindINFT` is ERC-721-compatible for wallet, marketplace, and indexer methods while keeping ERC-7857-style private metadata functions. TEE/ZKP verifier proof validation remains deferred.

## Direct Node Fallback

`contracts/scripts/mint-inft.ts` includes a direct storage-node fallback for cases where the indexer is reachable but the SDK submit path uses the old Flow ABI. The fallback manually submits the wrapped Flow transaction, waits for the storage node log entry, then uploads segments with the existing log entry. The 2026-05-01 run verified this path through storage upload and Galileo mint.
