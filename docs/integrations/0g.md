# 0G Integration Plan

## Targets

- 0G Compute for active inference.
- 0G Storage for agent state, memory, and run logs.
- 0G Chain for the minimal iNFT mint on Galileo testnet.

## Current Scaffold

- Contract network config points at 0G Galileo testnet by default through `ZERO_G_RPC_URL`.
- Seeded storage objects live in `data/snapshots/zerog-storage.seed.json`.
- The iNFT placeholder stores `storageUri`, `storageHash`, `model`, `strategyDigest`, and `aiq`.

## Official Reference Points

- Galileo testnet chain ID: `16602`.
- Development RPC placeholder: `https://evmrpc-testnet.0g.ai`.
- 0G Storage supports Go and TypeScript SDKs for upload/download workflows.
- 0G Compute exposes inference through web UI, CLI, SDK, and direct API access.

References:

- https://docs.0g.ai/developer-hub/testnet/testnet-overview
- https://docs.0g.ai/developer-hub/building-on-0g/storage/sdk
- https://docs.0g.ai/developer-hub/building-on-0g/compute-network/inference

## Integration Steps

1. Upload `agent-alpha` intelligence JSON to 0G Storage.
2. Read the uploaded object back and record the content hash.
3. Run one active inference call through 0G Compute and record provider, model, latency, and fallback status.
4. Mint `HivemindINFT` on Galileo with the verified storage URI and hash.
5. Add explorer and storage readback evidence to the demo notes.

## Safety

- Use funded testnet wallets only.
- Never commit `DEPLOYER_PRIVATE_KEY`, 0G bearer tokens, or provider secrets.
- Treat 0G addresses as testnet-drift-prone; verify addresses before final demo.
